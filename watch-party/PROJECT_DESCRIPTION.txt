1. О ПРОЕКТЕ
================================================================================

Watch Party — это веб-приложение и десктопное приложение для совместного
просмотра видео с друзьями в реальном времени. Основная идея: один человек
создаёт комнату, остальные присоединяются по коду, и все видят одно и то же
видео с синхронизацией воспроизведения (play/pause/seek).

Проект поддерживает два режима работы:
  — Браузерная версия (Vite + React) — доступна через Railway или статический
    хостинг. Работает YouTube-синхронизация и чат.
  — Десктопное приложение (Electron) — дополнительно включает встроенный
    браузер с перехватчиком сетевых запросов для обнаружения HLS/MP4
    видеопотоков с любых сайтов.


2. СТЕК ТЕХНОЛОГИЙ
================================================================================

Фронтенд:
  — React 19            — пользовательский интерфейс
  — React Router DOM v7 — маршрутизация (HashRouter для совместимости с
                           Electron)
  — Vite 8              — сборщик проекта
  — Tailwind CSS v4     — стилизация (утилитарный CSS)
  — Firebase JS SDK v12 — связь с бэкендом

Бэкенд (Firebase):
  — Firebase Authentication     — анонимная аутентификация пользователей
  — Firebase Realtime Database  — хранение данных комнат, пользователей,
                                  сообщений чата, синхронизация состояния
                                  плеера

Десктоп (Electron):
  — Electron 31                          — десктопная оболочка
  — @ghostery/adblocker-electron         — встроенный блокировщик рекламы
  — HLS.js                               — воспроизведение HLS-потоков
  — electron-builder                     — сборка установщика (.exe portable)

5. СИСТЕМА АУТЕНТИФИКАЦИИ
================================================================================

  На данный момент поддерживается только анонимный вход через Firebase Auth.

  Как это работает:
    1. При загрузке страницы хук useAuth подписывается на onAuthStateChanged
    2. Если пользователь не авторизован — автоматически выполняется
       signInAnonymously()
    3. При первом входе генерируется случайное имя
       (BravePanda42, CoolFox17 и т.д.) через updateProfile()
    4. Имя сохраняется в localStorage и восстанавливается при повторных
       визитах

  Планируется добавление регистрации и входа по email/паролю
  (см. plans/registration-login-plan.md).


6. СИСТЕМА КОМНАТ
================================================================================

  Каждая комната имеет:
    — Уникальный ID (генерируется случайная строка)
    — Название (задаётся при создании)
    — Тип: публичная (отображается на главной) или приватная (вход по паролю)
    — Пароль (только для приватных комнат)
    — Хост (создатель комнаты)
    — Список участников (members)
    — Счётчик участников (memberCount)
    — Текущее видео (currentVideoId)
    — Статус плеера (idle/playing/paused)
    — Последняя позиция (lastPosition)
    — Тип плеера (youtube/browser)
    — Чат (messages)

  Данные хранятся в Firebase Realtime Database:
    /rooms/{roomId}              — основные данные комнаты
    /rooms/{roomId}/members      — участники
    /rooms/{roomId}/messages     — сообщения чата
    /publicRooms/{roomId}        — индекс публичных комнат


7. СИНХРОНИЗАЦИЯ ВИДЕО
================================================================================

  YouTube-режим:
    — Один участник загружает видео по URL/ID
    — Firebase обновляет currentVideoId, status, lastPosition
    — Все участники подписаны на onValue() и синхронизируются автоматически
    — При play/pause/seek хост отправляет команды в Firebase, остальные
      повторяют

  Браузерный режим (только Electron):
    — Встроенный WebView2 загружает любой сайт
    — Preload-скрипт (browser-preload.js) сканирует страницу на наличие
      <video>-элементов и HLS-потоков
    — JS-сниффер (electron/video-sniffer.js) находит URL видео на странице
    — Найденные потоки отображаются в панели "Detected Video Streams"
    — HLS.js воспроизводит .m3u8 потоки прямо в приложении
    — Синхронизация через useBrowserSync.js


8. ЧАТ
================================================================================

  — Реализован через Firebase Realtime Database
  — Сообщения хранятся в /rooms/{roomId}/messages
  — Каждое сообщение содержит: uid, name, text, timestamp
  — Сортировка по timestamp
  — Автоматическая подписка через onValue()

10. РЕЖИМЫ ЗАПУСКА
================================================================================

  Режим разработки:
    npm run dev           — Vite дев-сервер (только фронтенд)
    start-dev.bat         — Полный цикл с Electron

  Сборка:
    npm run build              — Сборка фронтенда
    npm run build:railway      — Сборка для Railway (без Electron)
    npm run build:win          — Сборка + Electron установщик

  Railway (веб-версия):
    — Использует nixpacks.toml для сборки
    — VITE_RAILWAY=1 отключает Electron-плагины
    — Запускается через "npm run preview" на порту $PORT
